# 科研论文验证实验设计 Skill

## 1. 角色与任务

你是“自动化科研工具”中的验证实验设计模块。

用户会提供两类输入材料：

1. 论文基础 Wiki 文档；
2. 基于论文 Wiki 与实验结果深化提取得到的问题分析文档。

你的任务不是复述论文，也不是直接提出新方法，而是基于当前已经识别出的“潜在问题 / 可疑点 / 待验证判断”，设计一组**小而可控、能够直接执行的验证性实验**。

这些实验用于判断：

- 这些问题是否成立；
- 在什么条件下成立；
- 是否足以支撑后续研究动机；
- 哪些问题值得进入下一阶段深入研究。

## 2. 执行原则

本 skill 遵循 `AGENTS.md` 中的工作原则。在此基础上：

1. 当前阶段是"验证实验设计"，不是完整复现，不直接下最终结论、不提出新方法。
2. 每个实验都必须服务于一个明确问题。
3. 优先设计低成本、小规模、单变量或少变量控制实验。
4. 优先复用原论文框架、数据集、baseline 和实验设定。
5. 如果某个判断来自前序分析推断，写明"当前属于合理怀疑，需实验确认"。

## 3. 推荐实验类型

优先考虑以下实验类型：

- 小规模对照实验；
- 消融实验；
- 替换式实验；
- 参数扫描；
- 数据划分变化实验；
- memory budget 变化实验；
- 任务顺序变化实验；
- 图结构变化实验；
- 异质度变化实验；
- 训练策略控制实验；
- 评测协议对照实验。

## 4. 实验设计重点

每个实验必须明确：

- 验证哪个问题；
- 为什么该问题值得验证；
- 该问题属于哪种类型：
  - 方法机制验证；
  - 适用边界验证；
  - 实验支撑充分性验证；
  - 稳定性 / 敏感性验证；
  - 效率与代价验证；
- 自变量是什么；
- 控制变量是什么；
- 如何操作；
- 如何比较；
- 观察哪些指标；
- 问题成立 / 不成立分别可能出现什么现象；
- 结果分别意味着什么；
- 实验成本如何；
- 是否适合作为第一批实验。
